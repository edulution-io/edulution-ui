<task type="auto">
  <name>Task 2: Write add-feature.md workflow guide</name>
  <files>edulution-ai-framework/docs/workflows/add-feature.md</files>
  <action>
Write `docs/workflows/add-feature.md` -- an AI-consumable workflow guide for adding a feature to an existing edulution project.

Structure per CONTEXT.md locked decisions:

1. **Header section**: Title, purpose statement, prerequisite (working edulution project -- either scaffolded or manually set up).

2. **Step 1: Understand the project**
   - **Read:** `AGENTS.md`, `package.json`, list `src/` directory
   - Detect project type from `@` reference (custom-app or styled-page)
   - Detect available features (same detection logic as new-app.md step 2)
   - **Do not:** Ask the user what project type or features they have -- detect from files

3. **Step 2: Clarify the feature**
   - **Ask the user:** "What type of feature are you adding?"
     - 1. **New page/route** -- A new page with its own URL path (React custom-app only)
     - 2. **API integration** -- Connect to an edulution API endpoint and display/use the data
     - 3. **Page section** -- Add a new section to an existing page (HTML content + styling)
   - If project is styled-page and user picks "New page/route", explain this is React-only and suggest option 3 or upgrading to custom-app
   - **Do not:** Let the AI pick the feature type -- the user must choose

4. **Step 3: Feature details** (branches by feature type)

   **If New page/route (custom-app only):**
   - **Ask the user:** "What should this page be called? (e.g., Dashboard, Settings, UserList)"
   - **Ask the user:** "What is the URL path for this page? (e.g., /dashboard, /settings)"
   - **Read:** `edulution-ai-framework/custom-app/AGENTS.md` (re-read conventions)
   - Create the page component file, add route in App.tsx
   - Include `<!-- TODO: @ui-kit/ComponentName -->` markers and `// TODO: Replace with @edulution-io/ui-kit` comments where ui-kit components would be used (Button, Card, Input, etc.)
   - **Do not:** Use named exports -- custom-app AGENTS.md requires default exports
   - **Do not:** Import React hooks as `React.useState` -- import directly
   - **Do not:** Use inline styles -- use Tailwind classes with `cn()` if Tailwind is detected

   **If API integration:**
   - **Read:** `edulution-ai-framework/artifacts/swagger-spec.json`
   - **Ask the user:** "Which API endpoint do you want to integrate? (e.g., /api/users, /api/bulletins)"
   - Derive TypeScript types from the swagger spec response shapes -- no `any` types
   - **If custom-app:** Create API call in a Zustand store (not in component), use the existing apiClient
   - **If styled-page:** Use native fetch via the existing `src/api.js` wrapper
   - **Do not:** Define types manually when swagger spec has the response shape
   - **Do not:** Put API calls in React components -- use Zustand stores (custom-app AGENTS.md rule)

   **If Page section (works for both project types):**
   - **Ask the user:** "What should this section contain? (e.g., a data table, a form, an info card)"
   - **If custom-app:** Create a React component, import into existing page
   - **If styled-page:** Add a `<section>` in index.html with Tailwind utility classes
   - Include ui-kit TODO markers for components that would use ui-kit
   - **Read:** Project-type AGENTS.md layer before generating code
   - **Do not:** Use complex JS interactivity in styled pages -- suggest upgrading to custom-app if the section needs state management

5. **Step 4: Verify the feature**
   - Run dev server, visually confirm feature works
   - If tests are configured (vitest detected), suggest writing a test
   - Suggest a commit with descriptive message
   - **Check:** Feature renders without errors, follows conventions from AGENTS.md

Format using the same approved block structure as new-app.md. Keep total file under 250 lines. Target 180-220 lines (this guide has more branches).

Do not embed code snippets. Describe what to create, let the AI generate code following AGENTS.md conventions. Include ui-kit TODO markers as described in CONTEXT.md.
</action>
