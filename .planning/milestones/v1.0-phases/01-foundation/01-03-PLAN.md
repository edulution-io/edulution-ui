Create `.github/actions/checkout-submodule/action.yml` in the framework repo.

This is a composite action that consumer projects reference in their GitHub Actions workflows to authenticate and checkout the private framework submodule.

File content (based on research pattern):

```yaml
name: 'Checkout with edulution-ai-framework'
description: 'Checks out the consumer repo with the private edulution-ai-framework submodule authenticated and initialized'
inputs:
  token:
    description: 'GitHub fine-grained PAT with read access to the edulution-ai-framework repo (Contents: Read-only, Metadata: Read-only)'
    required: true
  fetch-depth:
    description: 'Number of commits to fetch (0 for full history)'
    required: false
    default: '1'
runs:
  using: 'composite'
  steps:
    - name: Checkout repository with submodules
      uses: actions/checkout@v5
      with:
        token: ${{ inputs.token }}
        submodules: 'recursive'
        fetch-depth: ${{ inputs.fetch-depth }}
```

Key design choices:

- Uses `actions/checkout@v5` (same version as edulution-ui's existing workflows)
- `submodules: 'recursive'` handles nested submodules if any
- Token input is required -- the PAT must have read access to the framework repo
- Optional `fetch-depth` for consumer flexibility (default shallow clone for speed)
- The token automatically configures HTTPS credentials for the submodule checkout

Commit:

```bash
cd edulution-ai-framework
git add .github/actions/checkout-submodule/action.yml
```
