Step 1 -- Create `typescript/AGENTS.md` (shared TypeScript conventions).

This file contains coding conventions that apply to both edulution-ui AND custom React app projects -- anywhere TypeScript is used. Content extracted from existing AGENTS.md lines 22-55 and CONVENTIONS.md:

**Coding Style section:**

- File names must match the default export name
- Never add comments in code (self-documenting through clear naming)
- Prefer default exports over named exports; `export default` at end of file
- Use const objects and derived types instead of enums
- Use the name "delete" over "remove"
- Use generic types over unsafe type casting

**React Conventions section:**

- Function components as arrow functions only
- Allow prop spreading
- Avoid `console` except `info`, `warn`, `error`
- Import React hooks at top of file (`import { useEffect } from 'react'`), never `React.useEffect` syntax
- Use `cn()` function for className composition (clsx + tailwind-merge)

**Naming Conventions section:**

- Functions: camelCase
- Components: PascalCase
- Variables: camelCase (local), UPPER_SNAKE_CASE (constants)
- Types/interfaces: PascalCase
- Prefix unused parameters with underscore (`_param`)

**Linting and Formatting section:**

- ESLint: Airbnb + TypeScript config with a11y and import rules
- Prettier: 2 spaces, 120 character width, single quotes, trailing commas, LF line endings
- Single attribute per line in JSX
- Tailwind CSS class sorting via prettier-plugin-tailwindcss

**Testing section (general):**

- Co-locate tests with source using `*.spec.ts(x)` or `test/` folders
- Keep tests fast and deterministic
- Add meaningful assertions

Target: 70-90 lines. MUST stay under 150.

Do NOT include: NestJS-specific patterns, Nx workspace details, eduApi, Zustand store patterns, edulution-ui file paths, build commands.
