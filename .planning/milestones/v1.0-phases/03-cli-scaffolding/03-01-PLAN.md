**src/prompts.ts** -- Interactive prompt flow using `@inquirer/prompts`:

Export a single `runPrompts()` async function that returns `ScaffoldConfig`.

1. **Project type** -- `select` prompt with two choices: "Custom React App" (value: `custom-app`) and "Styled HTML Page" (value: `styled-page`).

2. **Project name** -- `input` prompt with validation: required, lowercase alphanumeric with hyphens only (`/^[a-z0-9-]+$/`), must not start/end with hyphen. Creates `./{name}/` in cwd.

3. **Feature selection** -- `checkbox` prompt. Filter out `tests` feature when project type is `styled-page`. All features unchecked by default. Post-process: if `auth` is selected but `api-client` is not, auto-add `api-client` to the array (safety fallback for the auth->api dependency per Pitfall 1 in research).

4. **ui-kit mode** (only for `custom-app`) -- `select` prompt with two choices: "npm package (@edulution-io/ui-kit)" (value: `npm`) and "Local checkout (file: reference)" (value: `local`).

5. **ui-kit local path** (only if local mode selected) -- `input` prompt with default `../ui-kit`. Validate: check if `path.resolve(cwd, value)` exists using `fs.existsSync`. If not found, use `confirm` prompt: "Path not found. Continue anyway?". If user says no, re-prompt for path.

Return the assembled `ScaffoldConfig`.

**src/scaffold.ts** -- Core scaffold engine:

Export a single `scaffold(config: ScaffoldConfig)` async function.

Implementation follows the three-phase pipeline from research:

**Phase 1: Resolve template directory.** Determine the path to the templates directory. At runtime, templates live relative to the CLI package root. Use `import.meta.url` and `fileURLToPath` to resolve `../templates/{projectType}/` relative to the compiled output. The key insight: since package.json `files` includes `templates/`, the templates directory is a sibling of `dist/` in the published package. So from `dist/index.mjs`, templates are at `../templates/`.

**Phase 2: Copy base template.** Use `fs.cp(baseDir, targetDir, { recursive: true })` where `baseDir` is `templates/{projectType}/base/` and `targetDir` is `path.resolve(cwd, config.projectName)`. Check that targetDir does not already exist before copying; throw error if it does.

**Phase 3: Overlay selected features.** For each feature in `config.features`, copy `templates/{projectType}/features/{feature}/` into `targetDir`. Use a custom copy function that skips `package.json` files during the overlay copy (they are handled in the merge phase). All other files overwrite or add to the target.

**Phase 4: Deep-merge package.json.** Read `package.json` from base. For each selected feature that has a `package.json` in its feature directory, read it. Use `deepmerge` with `arrayMerge: overwriteArrayMerge` (source array replaces target array) to merge all package.json files. Set `merged.name = config.projectName`. Write the merged result to `targetDir/package.json`.

**Phase 5: Handle ui-kit wiring** (only for `custom-app`):

- If npm mode: add `"@edulution-io/ui-kit": "^1.0.0"` to dependencies in the merged package.json
- If local mode: add `"@edulution-io/ui-kit": "file:{relativePath}"` to dependencies where `relativePath` is computed from `targetDir` to `config.uiKitLocalPath`. Also add tsconfig paths alias: `"@edulution-io/ui-kit": ["{relativePath}/src"]` in the project's tsconfig.json `compilerOptions.paths`.

**Phase 6: Rename \_prefix files.** Walk the target directory recursively. For each file whose basename is in `RENAME_MAP`, rename it to the mapped name (e.g., `_gitignore` -> `.gitignore`).

**Phase 7: Replace placeholders.** Walk the target directory recursively. For each file with an extension in `TEXT_EXTENSIONS` (or filename starting with `.env`), read the file as UTF-8, replace all occurrences of each key in `PLACEHOLDER_DEFAULTS` with its value (override `{{PROJECT_NAME}}` with `config.projectName`), write back.

**Phase 8: Generate CLAUDE.md and AGENTS.md.** Write `CLAUDE.md` to targetDir containing `@edulution-ai-framework/AGENTS.md` (pointing to the submodule's base layer). Write `AGENTS.md` to targetDir containing `@edulution-ai-framework/{projectType}/AGENTS.md` (pointing to the project-type layer). These files enable AI tool convention discovery.

Use `picocolors` for colored console output during each phase (green checkmarks for completed steps).

**src/post-scaffold.ts** -- Post-scaffold automation:

Export a single `postScaffold(config: ScaffoldConfig)` async function.

Use `child_process.execSync` for git commands and `child_process.spawn` for npm install (to stream output).

**Step 1: git init.** Run `git init` in the project directory. Print green checkmark.

**Step 2: Add framework submodule.** Run `git submodule add {FRAMEWORK_REPO_URL} edulution-ai-framework` in the project directory. Wrap in try/catch: if it fails (SSH key issue), print a yellow warning with the error message, suggest the user can manually add the submodule later with the HTTPS URL alternative: `git submodule add https://github.com/edulution-io/edulution-ai-framework.git edulution-ai-framework`. Continue execution even if this step fails.

**Step 3: npm install prompt.** Use `confirm` prompt from `@inquirer/prompts`: "Run npm install?" (default: yes). If yes, spawn `npm install` in the project directory with `stdio: 'inherit'` so the user sees output. Wait for completion. Print green checkmark on success, yellow warning on failure.

**Step 4: Initial commit.** Run `git add -A` then `git commit -m "chore: scaffold {projectName} with edulution-create"` in the project directory. Only attempt if git init succeeded.

**Step 5: Print summary.** Use `picocolors` for formatted output:

- Header: "Project {projectName} created successfully!"
- Feature checklist: For each possible feature, print a green checkmark if included or a grey dash if not selected. Format: `[x] Tailwind + edulution theme` or `[ ] API client (axios)`
- ui-kit mode (if custom-app): Show npm or local mode with path
- Next steps box:
  ```
  Next steps:
    cd {projectName}
    npm run dev
  ```
- For styled pages, show `npm run build:css` instead of `npm run dev` (or whatever the appropriate dev command is based on the template).

**src/index.ts** -- CLI entry point:

Simple orchestration wiring:

```typescript
#!/usr/bin/env node
import { runPrompts } from './prompts.js';
import { scaffold } from './scaffold.js';
import { postScaffold } from './post-scaffold.js';

const run = async () => {
  const config = await runPrompts();
  await scaffold(config);
  await postScaffold(config);
};

run().catch((error) => {
  console.error(error.message);
  process.exit(1);
});
```

Note the `.js` extensions in imports -- required for ESM module resolution with Node16 moduleResolution.
