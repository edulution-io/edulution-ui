<task type="auto">
  <name>Task 2: Update sync-framework.yml to dog-food composite action</name>
  <files>.github/workflows/sync-framework.yml</files>
  <action>
Update sync-framework.yml to use the framework's composite checkout action instead of plain actions/checkout for the edulution-ui source checkout. This dog-foods the framework's own CI helper.

Key constraint from research: The composite action (checkout-submodule) does NOT support a `path:` input. It only accepts `token` and `fetch-depth`. It checks out to the default working directory (`.`), not to a subdirectory. The current workflow checks out edulution-ui to `source/` path.

Changes needed in BOTH sync-swagger and sync-styling jobs:

1. Replace the "Checkout edulution-ui" step:
   BEFORE:
   - name: Checkout edulution-ui
     uses: actions/checkout@v5
     with:
     path: source
     persist-credentials: false

   AFTER:
   - name: Checkout edulution-ui with framework submodule
     uses: edulution-io/edulution-ai-framework/.github/actions/checkout-submodule@main
     with:
     token: ${{ steps.app-token.outputs.token }}

2. Update all `source/` path references in the sync steps to use `.` (current directory) since the composite action checks out to the working directory root:
   - sync-swagger job: `source/swagger-spec.json` becomes `swagger-spec.json`
   - sync-styling job: `source/apps/frontend/tailwind.config.ts` becomes `apps/frontend/tailwind.config.ts`, etc.

3. The "Checkout framework repo" step (target) stays unchanged -- it still uses actions/checkout@v5 with the framework repo and path: target.

4. The detect-changes job stays unchanged -- it uses a simple actions/checkout@v5 without path, which is fine (it only needs the file tree for path filtering, no submodule needed).

5. The app-token step must be moved BEFORE the composite action step in each sync job (it already is -- no reordering needed). The composite action uses the same app token for both submodule checkout (read) and artifact push (write) -- single auth flow per locked decision.
   </action>
