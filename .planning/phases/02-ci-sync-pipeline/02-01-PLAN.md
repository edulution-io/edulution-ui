  <action>
Create `.github/workflows/sync-framework.yml` with the following structure:

**Workflow trigger:**

- `on.push.branches: [dev]` with `paths` listing all 4 source files (swagger-spec.json, apps/frontend/tailwind.config.ts, libs/ui-kit/tailwind.config.ts, apps/frontend/src/index.scss)
- Add `workflow_dispatch` for manual testing

**Job 1: detect-changes**

- runs-on: ubuntu-24.04, timeout-minutes: 5
- Checkout edulution-ui with `actions/checkout@v5`
- Use `dorny/paths-filter@v3` to produce two boolean outputs: `swagger` and `styling`
  - swagger filter: `swagger-spec.json`
  - styling filter: `apps/frontend/tailwind.config.ts`, `libs/ui-kit/tailwind.config.ts`, `apps/frontend/src/index.scss`

**Job 2: sync-swagger**

- needs: detect-changes
- if: `needs.detect-changes.outputs.swagger == 'true' || github.event_name == 'workflow_dispatch'`
- runs-on: ubuntu-24.04, timeout-minutes: 10
- Steps:
  1. Generate App token via `actions/create-github-app-token@v1` with `app-id: ${{ vars.VERSION_BUMPER_APPID }}`, `private-key: ${{ secrets.VERSION_BUMPER_SECRET }}`, `owner: ${{ github.repository_owner }}`, `repositories: edulution-ai-framework`
  2. Checkout edulution-ui into `source/` with `persist-credentials: false`
  3. Checkout `edulution-io/edulution-ai-framework` into `target/` using the App token, ref: main
  4. Sync step: `mkdir -p target/artifacts`. Guard against empty file: `if [ ! -s source/swagger-spec.json ]` then skip. Copy file. `cd target && git diff --quiet -- artifacts/swagger-spec.json` to detect changes. Output `changed=true/false`.
  5. Generate diff summary step (only if changed=true): Use `git diff --stat` for the stat line. For swagger specifically, compare old vs new endpoint paths using `jq -r '.paths | keys[]'` on both the staged version and source, counting added/removed endpoints. Build multiline commit message with source SHA reference.
  6. Commit and push step (only if changed=true): Configure bot git user. `git add artifacts/swagger-spec.json`. `git pull --rebase origin main`. Commit with the generated summary message. `git push origin main`.

**Job 3: sync-styling**

- needs: detect-changes
- if: `needs.detect-changes.outputs.styling == 'true' || github.event_name == 'workflow_dispatch'`
- runs-on: ubuntu-24.04, timeout-minutes: 10
- Steps: Same pattern as sync-swagger but:
  1. Same App token generation
  2. Same dual checkout
  3. Sync step copies 3 files with rename mapping:
     - `source/apps/frontend/tailwind.config.ts` -> `target/artifacts/tailwind.config.ts`
     - `source/libs/ui-kit/tailwind.config.ts` -> `target/artifacts/tailwind.base.config.ts`
     - `source/apps/frontend/src/index.scss` -> `target/artifacts/index.scss`
       Then `git diff --quiet -- artifacts/tailwind.config.ts artifacts/tailwind.base.config.ts artifacts/index.scss`
  4. Generate diff summary: `git diff --stat` for the changed styling files. Build multiline commit message listing which files changed.
  5. Commit and push (only if changed): same bot config, `git add artifacts/tailwind.config.ts artifacts/tailwind.base.config.ts artifacts/index.scss`, pull rebase, commit, push.

**Important details from research (avoid pitfalls):**

- Use `persist-credentials: false` on the source (edulution-ui) checkout to prevent credential conflicts with the framework repo checkout
- Use `mkdir -p target/artifacts` before any copy (directory may not exist yet)
- Always `git pull --rebase origin main` before push to handle concurrent merge race conditions
- For `workflow_dispatch`, both sync jobs should run regardless of paths-filter output (the `|| github.event_name == 'workflow_dispatch'` condition)
- Use `jq` for swagger endpoint diff -- jq is pre-installed on ubuntu-24.04 runners
