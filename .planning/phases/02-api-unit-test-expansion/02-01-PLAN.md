**AuthGuard spec** (`apps/api/src/auth/auth.guard.spec.ts`):

Read `apps/api/src/auth/auth.guard.ts` to understand the exact implementation.

Key behaviors to test:

- `jest.mock('fs', ...)` at module level to mock readFileSync returning a fake RSA public key
- Mock JwtService with `verifyAsync` as jest.fn()
- Mock Reflector with `get` as jest.fn()
- Use createMockExecutionContext for all tests

Test cases:

- Valid token: verifyAsync resolves with user payload -> canActivate returns true, request.user is set
- Missing token on protected route: no authorization header -> throws UnauthorizedException
- Expired token: verifyAsync rejects with TokenExpiredError -> throws UnauthorizedException
- Malformed JWT structure: token is not a valid JWT string (e.g., "not.a.jwt") -> verifyAsync rejects -> throws UnauthorizedException
- Wrong algorithm token: verifyAsync rejects with algorithm mismatch error -> throws UnauthorizedException
- Public route bypass: Reflector.get returns true for PUBLIC_ROUTE_KEY -> canActivate returns true without token validation
- Token in authorization header extracted correctly: verify "Bearer xxx" -> "xxx" is passed to verifyAsync

**AccessGuard spec** (`apps/api/src/auth/access.guard.spec.ts`):

Read `apps/api/src/auth/access.guard.ts` to understand the implementation.

Key behaviors to test:

- Import and call `clearAccessCache()` from `apps/api/src/common/guards/accessCache` in beforeEach
- Mock Reflector, AppConfigService (appAccessMap), GlobalSettingsService (getAdminGroupsFromCache)
- Use createMockExecutionContext and createJwtUser

Test cases:

- Admin bypass: user ldapGroups includes admin group -> returns true regardless of app access
- Group match: user ldapGroups matches app access requirement -> returns true
- No matching groups: user ldapGroups do not match -> throws ForbiddenException (403)
- No user on request: request.user is undefined -> throws UnauthorizedException (401)
- Access cache hit: second call with same user/app returns cached result without calling services again
- Access cache miss after clear: clearing cache forces re-evaluation
- No APP_ACCESS_KEY metadata: no restriction set -> returns true (default open)

Include project license header on all new files.
