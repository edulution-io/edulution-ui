Output: 3 pipe spec files and 5 filter spec files with behavioral assertions covering valid/invalid inputs and all error branches.
</objective>

  - Empty string: check implementation (likely throws BadRequestException)
  - Null/undefined input: check implementation for behavior
  - Already parsed object (non-string input): check if pipe handles this case

  - Array with all matching entries: returns full array
  - Check how the pipe receives the user context (likely from request)
