Create `edulution-ui/AGENTS.md` in the framework repo. This layer is for developers working on the edulution-ui monorepo itself. It contains conventions SPECIFIC to the core project that do NOT apply to custom apps or styled pages.

Content to include, extracted from existing AGENTS.md and CONVENTIONS.md:

**Project Structure section:**

- Nx monorepo with apps/frontend (React + Vite), apps/api (NestJS), libs (shared TS utilities and UI modules)
- Shared code lives under `libs/` -- move utility functions, types, and constants there (in separate files). Do not move UI components to libs.
- scripts, docs, swagger-spec.json: helper scripts, documentation, and OpenAPI spec
- Path aliases: `@/*` -> `apps/frontend/src/*`, `@libs/*` -> `libs/src/*`, `@edulution-io/ui-kit` -> `libs/ui-kit/src/index.ts`

**Build, Test, and Development Commands section (from existing AGENTS.md lines 13-20):**

- Install: `npm install`
- Start frontend: `npm run dev` (Vite on http://localhost:5173)
- Start API: `npm run api` (Nest on http://localhost:3001)
- Both: `npm run serveFrontendAndApi`
- Build: `npm run build`, `npm run build:api`, `npm run build:all`
- Tests: `npm run test` (API/Jest), `npm run test:frontend` (Vitest)
- Lint: `npm run lint` (full) or `npx eslint path/to/file.ts --quiet` (single file, preferred)
- Format: `npm run format`
- Local infra: `docker compose up -d` (MongoDB, Redis)

**NestJS Conventions section (from existing AGENTS.md lines 40-48):**

- Use static Logger calls with service name as context: `Logger.log('message', ServiceName.name)`. Do NOT create instance loggers with `private readonly logger = new Logger()`.
- Use `CustomHttpException` for all HTTP errors (pass ErrorMessage i18n key, HttpStatus, optional data, optional domain)
- Migrations MUST always increase the schema version number
- File naming: kebab-case with suffix (`.service.ts`, `.controller.ts`, `.module.ts`, `.schema.ts`)
- DTOs: kebab-case with `.dto.ts` suffix in `dto/` subdirectory
- Use `class-validator` and `class-transformer` for DTO validation

**Frontend-Specific Patterns section (core-only patterns):**

- Use `eduApi` (axios) from `@/api/eduApi` for all API calls instead of native `fetch`
- Place API calls in Zustand store actions, NOT in components
- Use `ResponseType.BLOB` for blob responses and `handleApiError` for error handling
- All Zustand stores must have a `reset()` method and be registered in `cleanAllStores.ts`
- Use axios built-in `params` option instead of manual URLSearchParams
- Use SH wrapper components for shadcn/@radix-ui (e.g., `AccordionSH`, `DropdownMenuSH`, `CardSH`)
- Use only `@fortawesome/free-solid-svg-icons` for icons

**Testing section (edulution-ui specifics):**

- Frontend/libs: Vitest; setup at `apps/frontend/test/vitest.setup.ts`
- API: Jest; config at `apps/api/jest.config.ts`

**License Header section:**

- Every `.ts` and `.tsx` file MUST include the Netzint GmbH AGPL/commercial dual-license header (18 lines). Enforced by pre-commit hook.

**Pre-Commit Checks section:**

- Circular dependency detection, translation key completeness, filename validation, license header, Prettier formatting, ESLint staged

Target: 90-120 lines. MUST stay under 150.

Do NOT include: Design tokens (base layer), general TS coding style (typescript layer), general React conventions (typescript layer), git/PR conventions (base layer).
