  <action>
Add a new `e2e` job to the existing GitHub Actions workflow `.github/workflows/build-and-test.yml`.

Add the job AFTER the existing `test-frontend` job. The e2e job:

1. **Job definition:**

   ```yaml
   e2e:
     runs-on: ubuntu-24.04
     needs: prepare
     timeout-minutes: 30
     continue-on-error: true # Non-blocking per locked decision — starts as informational
     steps:
   ```

2. **Steps:**
   - Checkout code (same as other jobs)
   - Setup Node.js 22.x
   - Restore cached node_modules
   - Install Playwright browsers:
     ```yaml
     - name: Install Playwright browsers
       run: npx playwright install --with-deps chromium firefox webkit
     ```
   - Run E2E tests:
     ```yaml
     - name: Run E2E tests
       run: npm run test:e2e
       env:
         E2E_BASE_URL: ${{ secrets.E2E_BASE_URL }}
         E2E_ADMIN_USER: ${{ secrets.E2E_ADMIN_USER }}
         E2E_ADMIN_PASS: ${{ secrets.E2E_ADMIN_PASS }}
         E2E_TEACHER_USER: ${{ secrets.E2E_TEACHER_USER }}
         E2E_TEACHER_PASS: ${{ secrets.E2E_TEACHER_PASS }}
         E2E_STUDENT_USER: ${{ secrets.E2E_STUDENT_USER }}
         E2E_STUDENT_PASS: ${{ secrets.E2E_STUDENT_PASS }}
         E2E_PARENT_USER: ${{ secrets.E2E_PARENT_USER }}
         E2E_PARENT_PASS: ${{ secrets.E2E_PARENT_PASS }}
     ```
   - Upload Playwright report on failure (per CICD-03 requirement):
     ```yaml
     - name: Upload Playwright report
       uses: actions/upload-artifact@v4
       if: failure()
       with:
         name: playwright-report
         path: apps/e2e/playwright-report/
         retention-days: 14
     ```

3. **Key points:**
   - `continue-on-error: true` makes the job non-blocking (per locked decision: starts informational, promote to required once stable)
   - `needs: prepare` ensures dependencies are installed first
   - `timeout-minutes: 30` gives enough time for all browser tests with retries
   - Secrets are passed via `env` section (per locked decision: credentials stored in GitHub Actions secrets)
   - Playwright report uploaded ONLY on failure (per CICD-03)
   - `retention-days: 14` keeps reports for 2 weeks
     </action>

     </verify>
