**useKeyboardNavigation.spec.ts (FEHK-02):**
This is the most complex hook (250 lines). It manages keyboard navigation state for the file browser.

Setup:

- Mock useFileEditorStore to return { isFilePreviewVisible: false, setIsFilePreviewVisible: vi.fn(), currentlyEditingFile: null, resetCurrentlyEditingFile: vi.fn() }
- Mock isValidFileToPreview from '@libs/filesharing/utils/isValidFileToPreview' to return true
- Create mock files array: `[{ filePath: '/file1.txt', filename: 'file1.txt' }, { filePath: '/file2.txt', filename: 'file2.txt' }, ...]` matching DirectoryFileDTO shape
- Create mock onFileOpen callback as vi.fn()

Test cases using renderHook + fireEvent.keyDown on window:

- **ArrowDown (list view):** focusedIndex increments by 1, isKeyboardNavActive becomes true
- **ArrowUp (list view):** focusedIndex decrements by 1
- **ArrowDown at last item:** focusedIndex stays at last index (clamped)
- **ArrowUp at first item:** focusedIndex stays at 0 (clamped)
- **Enter on focused file:** calls onFileOpen with focusedFile
- **Escape when keyboard nav active:** resets focusedIndex to null, isKeyboardNavActive to false
- **Escape when file preview visible:** calls setIsFilePreviewVisible(false) instead of reset
- **ArrowRight/ArrowLeft in grid view:** navigates by 1 item
- **ArrowDown/ArrowUp in grid view:** navigates by gridItemsPerRow
- **No navigation when isEnabled is false**
- **No navigation when target is input/textarea element** (check the isInputElement guard)
- **handleItemClick:** sets focusedIndex to file's index, calls onFileOpen, sets isKeyboardNavActive to false
- **resetNavigation:** sets focusedIndex to null and isKeyboardNavActive to false
- **Files change:** resets focusedIndex and isKeyboardNavActive

Use `act()` wrapper from @testing-library/react when dispatching events:

```typescript
act(() => {
  fireEvent.keyDown(window, { key: 'ArrowDown' });
});
```

**useNotifications.spec.ts (FEHK-03):**
This is an orchestrator hook that sets up many SSE listeners. It has many dependencies. Strategy: mock most sub-hooks and verify the SSE event handlers call correct store actions.

Setup mocks:

```typescript
vi.mock('@/hooks/useLdapGroups', () => ({ default: () => ({ isSuperAdmin: false, isAuthReady: true }) }));
vi.mock('@/hooks/useLanguage', () => ({ default: () => ({ language: 'en' }) }));
vi.mock('@/hooks/useIsAppActive', () => ({ default: (app: string) => true }));
vi.mock('@/hooks/useDockerContainerEvents', () => ({ default: () => {} }));
vi.mock('@/hooks/useSseHeartbeatMonitor', () => ({ default: () => {} }));
vi.mock('@/hooks/useDownloadProgressToast', () => ({ default: () => {} }));
vi.mock('@/hooks/useFileOperationProgressToast', () => ({ default: () => {} }));
vi.mock('@/pages/FileSharing/hooks/useFileOperationProgress', () => ({ default: () => {} }));
vi.mock('@/hooks/useOrganizationType', () => ({ default: () => ({ isSchoolEnvironment: true }) }));
```

Pre-populate stores:

- Set useSseStore with a MockEventSource
- Set useMailsStore, useConferenceStore, useSurveyTablesPageStore with mock data

Test strategy: Since useNotifications uses useSseEventListener internally, and useSseEventListener uses the EventSource from useSseStore, we can test by:

1. Render the hook
2. Dispatch events on the MockEventSource
3. Assert that the correct store actions were called

Key tests:

- **Initial load:** When isAuthReady, calls getMails (if mail active and not superAdmin), updateOpenSurveys (if surveys active), getConferences (if conferences active), fetchUnreadCount
- **MAIL_NEW_MAIL event:** Dispatching this event on MockEventSource calls getMails and shows toast
- **MAIL_FLAGS_CHANGED event:** Calls getMails
- **CONFERENCE_CREATED event:** Adds conference to store
- **CONFERENCE_STARTED/STOPPED events:** Updates conference isRunning state
- **CONFERENCE_DELETED event:** Removes conferences from store
- **SURVEY_CREATED/UPDATED/DELETED events:** Calls updateOpenSurveys
- **BULLETIN_UPDATED event:** Calls addBulletinBoardNotification
- **APPCONFIG_UPDATED event:** Calls getAppConfigs and getPublicAppConfigs

Wrap in renderWithProviders for router + i18n context.

This is a complex test -- if the dependency mocking becomes unwieldy, prioritize testing the initial load effect and 2-3 SSE event handlers rather than all of them. The goal is behavioral coverage of the hook's core responsibilities.
